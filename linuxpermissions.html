Linux Permissions

Skills: Operating Systems, Linux, Virtual Machines

Scenario

Using a Linux lab with a fictional organization, I will review the file permissions for certain files and directories in the projects directory. The permissions will not adhere to the level of authorization required by the organization. Checking and updating permissions will help keep the system secure and will require the following tasks:

Check file and directory details

First I reviewed the current state of permissions in the projects directory. The below screenshot shows the command I used on the first line and then the output as a result of the command. I used the ls command and the -la option to display a listing of file contents that also returns hidden files. The output indicates there is one directory named drafts, one hidden file named .project_x.txt and four other project files. The 10-character string in the first column indicates the permissions for the directory and each file.

Describe the permissions string

The permissions string is 10 characters long with the first character indicating the file type. if the character is a d, it is a directory. If the character is a hyphen (-), it's a regular file. The remaining characters indicate the permissions for the 3 distinct owner types: user, group, and other. The user type is the owner of the file, and the group type is a group the user belongs to in the company's organization. Anyone with access to the system that is not part of the group or the user belongs to the other category.

Each owner type has 3 permissions turned on or off. The first being read, indicated by an r, meaning the owner type can read the file contents. The second is write, indicated by a w, meaning the owner type can make modifications to the file contents or add new files to a directory. The last permission is execute, indicated by an x, meaning the owner type can execute the file if it's a program or access a directory. If an owner type has a permission turned off, it will be indicated by a hyphen (-) instead of a letter.

Using the Linux lab for an example, the line -rw-rw-r-- 1 researcher2 research_team 46 Aug 30 22:04 project_t.txt is in reference to the file project_t.txt. I know it's a regular file because the first character is a hyphen (-). The user permissions are the 2nd, 3rd, and 4th characters (rw-). The group permissions are the 5th, 6th, and 7th characters (rw-) and match the permissions of the user. The other permissions are the last 3 characters and the r-- indicate anyone not in the group or the user types can only read the file.

Change file permissions

The organization determined the other owner type shouldn't have write access to any files. Upon review of the current permissions, I determined project_k.txt must have the write access removed for other.

Using the chmod command to change permissions, I indicated two arguments: the first being the permissions to be changed (in this case o-w for other owner type minus write permissions). The second argument specifies the file to be modified, which is project_k.txt. To confirm the updates had been made, I used ls -la to output the files and directory list with permissions again.

Change file permissions on a hidden file

The hidden file .project_x.txt had been archived and the organization did not want anyone to continue to have write access to this project. Reviewing the permissions, I determined both the user and group owner types still retained write permissions and the group did not have read permissions.

While I could use minus (-) and plus (+) to remove and add permissions for each owner type as needed, I decided to use equal (=) to overwrite the current permissions with the correct permissions. This allowed me to complete the changes to the group type without using two separate modification arguments (g-w,g+r). Either way works, but the equal made the command cleaner. I used the chmod command with the arguments u=r,g=r and .project_x.txt. I then used the command ls -la to review the permissions and confirm my changes had been made.

Change directory permissions

The organization wants to restrict access to the drafts directory and its contents to only the owner, researcher2. The current permissions indicate the group has execute permissions, meaning other users in the same group as the owner researcher2 have access to the drafts directory and its files. Because the user researcher2 has all appropriate permissions, I only needed to use the chmod command to remove the group owner type’s permission.

Summary

In a scenario where a fictional organization needed a review of the level of authorizations of files and directories in the projects directory to ensure they met requirements set by the organization’s standards, I needed to modify multiple permissions. The first step was using ls -la to check the current permissions for the directory. I then used the chmod command multiple times to change the permissions on files and directories in the projects directory to match the appropriate permissions needed.
